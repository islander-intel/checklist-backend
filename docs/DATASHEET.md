# checklist-backend — Datasheet

> **API Contract & Service Specification** > Reference document for the **simply todo** logic tier (Worker).

---

## Service Overview

The `checklist-backend` is a high-performance Go service responsible for the CRUD (Create, Read, Update, Delete) lifecycle of tasks. It acts as the exclusive gateway to the `todo-db` vault.

| Attribute | Value |
|---|---|
| **Language/Runtime** | Go 1.22 (Gin Framework) |
| **Service Port** | `8001` |
| **Base Path** | `/api/v1` |
| **Auth Strategy** | `X-Internal-Token` (Shared Secret) |
| **Primary Data Source** | `todo-db` (Postgres 15) |

---

## Network Topology

This service is internal. It is **not** designed to be exposed directly to the public internet.

1.  **Ingress:** Receives traffic from `todo-router` on port `8001`.
2.  **Egress:** Sends traffic to `postgres-db` on port `5432`.

---

## Security Protocol

All requests to this worker must include the following header for the internal handshake:

```http
X-Internal-Token: [INTERNAL_AUTH_TOKEN]
```

*Requests missing this token or providing an invalid one will return `401 Unauthorized`.*

## API Endpoints

### 1. Create Task
`POST /tasks`

**Request Body:**
```json
{
  "title": "String (Required)",
  "description": "String (Optional)",
  "is_completed": "Boolean (Default: false)"
}
```

**Success Response (`201 Created`):**
```json
{
  "id": "uuid-string",
  "title": "String",
  "description": "String",
  "is_completed": false,
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

---

### 2. List All Tasks
`GET /tasks`

**Success Response (`200 OK`):**
```json
[
  {
    "id": "uuid-string",
    "title": "String",
    "description": "String",
    "is_completed": false,
    "created_at": "timestamp",
    "updated_at": "timestamp"
  }
]
```

---

### 3. Update Task
`PUT /tasks/:id`

**Request Body:**
```json
{
  "title": "String",
  "description": "String",
  "is_completed": "Boolean"
}
```

**Success Response (`200 OK`):**
```json
{
  "status": "updated",
  "updated_at": "timestamp"
}
```

---

### 4. Delete Task
`DELETE /tasks/:id`

**Success Response (`204 No Content`):**
*(No body returned)*

## Data Dictionary (Model Mapping)

These fields align with the `todo-db` schema and the `internal/models/task.go` struct.

| Field | Type | Constraint | Description |
|---|---|---|---|
| `id` | `UUID` | Read-only | Generated by Database Vault. |
| `title` | `String` | 1-255 chars | Primary task name. |
| `description` | `String` | Nullable | Detailed notes. |
| `is_completed` | `Boolean` | Default: `false` | Completion toggle. |
| `created_at` | `DateTime` | Read-only | UTC timestamp of creation. |
| `updated_at` | `DateTime` | Read-only | UTC timestamp (managed by DB trigger). |

---

## Error Handling

| Status Code | Meaning | Cause |
|---|---|---|
| `400` | Bad Request | Missing `title` or invalid JSON body. |
| `401` | Unauthorized | Missing or incorrect `X-Internal-Token`. |
| `404` | Not Found | Task ID does not exist in the vault. |
| `500` | Internal Error | Database connection failure or query timeout. |

## Maintenance Notes

* **Timestamps:** All time values are returned in ISO 8601 (UTC).
* **Concurrency:** The backend utilizes a connection pool (`pgxpool`). High-frequency polling is supported.
* **Logging:** Every CRUD action is logged with a trace ID in the `checklist-backend` container logs.
